## 典型用户画像

## 我们的定位：AI 内容经济的"合规中间件"

```
任何 AI 内容工具
    │
    ├─ 需要翻译？ → 调 translate API
    ├─ 需要审查？ → 调 compliance-check API
    ├─ 需要 SEO？  → 调 seo-optimize API
    ├─ 需要知道平台规则？ → 读 platform-rules.json
    └─ 需要审查报告？ → 调 compliance-report API
```

**我们不和 AI 工具竞争——我们成为它们的一部分。**

## 核心护城河

1. **17 平台规则数据库** — 数据壁垒，越积累越难复制
2. **中国广告法 + 多平台规则** — 法律 + 技术双重壁垒
3. **中英双语能力** — 跨境场景天然优势
4. **多形态分发** — API + CLI + MCP + VS Code + Chrome，全渠道覆盖
5. **X402 原生集成** — AI Agent 可以自动发现、自动付费调用

## 推理逻辑

当一个 AI 新手说"帮我做一个小红书工具"时：

```
用户的 AI 助手 → 生成小红书内容
                    ↓
              需要检查合规 → 发现我们的 API
                    ↓
              调 compliance-check（platform=xiaohongshu）
                    ↓
              返回审查结果 → 修改 → 发布
```

**我们的角色：** 不是那个做小红书工具的人——是那个所有做小红书工具的 AI 都要调的后端。
